两种本地预览，别搞混

调不同的东西，要开不同的终端

终端 A：live-server（端口 8080）MacBook 预览客户看到的报价页面
看的是 pkg/ 目录下的 HTML 文件
启动命令：live-server --port=8080
浏览器打开：localhost:8080/pkg/my-jason-400m/latest.html
什么时候开：改样式、改排版、改模板的时候
终端 B：Flask（端口 5050）MacBook 预览业务员后台 / 工作台
看的是你现在正在看的这个页面
启动命令：python3 build/admin_server.py
浏览器打开：localhost:5050
什么时候开：改后台功能、改工作台页面的时候
两个互不影响。你可以只开一个，也可以同时开两个，取决于你在调什么。

项目里有哪些文件？分别是干嘛的？

文件中文说明改完怎么生效
static/style.css 客户页面的样式（颜色、字体、间距、排版） wrangler 部署
static/*.js 客户页面的交互功能（3D场景、语言切换、产品详情弹窗） wrangler 部署
build/templates/base.html 客户页面的HTML模板（页面骨架结构，产品卡片怎么排列） 先生成再 wrangler 部署
build/transform.py 数据处理脚本（飞书原始数据→网页能用的格式，算价格、算运费） 先生成再 wrangler 部署
build/generator.py 页面生成器（拉飞书数据 + 套模板 = 生成客户HTML页面） Mac mini 后台调用
（业务员后台点生成网页触发，链路修复中）
build/feishu_client.py 飞书连接器（负责从飞书多维表格读写数据，后台和生成器都会用到） Mac mini 运行 MacBook 也用
build/admin_server.py 业务员后台程序（就是 crm.provenlift.com 背后跑的程序） git 推到 Mac mini
build/admin_templates/*.html 后台页面模板（工作台、业务员后台、拍摄管理的页面） git 推到 Mac mini
pkg/xxx/latest.html 生成好的客户页面（generator.py 自动输出到这里） wrangler 部署
static/og-images/*.jpg WhatsApp 分享缩略图（发链接时显示的产品大图） wrangler 部署

保存就能看到效果 vs 需要跑生成命令

✅ 保存后自动刷新（不用跑命令） 改了这些文件，保存后 live-server 自动刷新浏览器：
· static/style.css（样式）
· static/*.js（交互功能）
因为这些文件被客户页面直接引用，改了就是改了。
⚠️ 需要重新生成（跑一次命令） 改了这些文件，需要重新跑生成器才能看到效果：
· build/templates/base.html（模板骨架）
· build/transform.py（数据处理）
· 飞书里的产品数据
因为这些内容要先"拼装"成 HTML 才能看到。
生成命令：cd ~/Projects/gym-quote-page && python3 build/generator.py

完整工作流（从开机到关机）

你最常做的三种事情，分别怎么操作

🔧 修复客户网页的排版 / 样式 / 数据显示问题
1

开终端，启动本地预览 MacBook

cd ~/Projects/gym-quote-page && live-server --port=8080

浏览器打开 localhost:8080/pkg/my-jason-400m/latest.html

这个终端在调试期间保持开着。你随时可以在浏览器里看效果。
2

改文件，看效果 MacBook

用编辑器改对应文件（参考上面的文件说明表）。改样式直接保存看效果；改模板或数据处理则需要跑一次生成命令。

反复修改直到本地效果满意为止。

3

部署到线上，让客户看到 → Cloudflare 云端

cd ~/Projects/gym-quote-page && export CLOUDFLARE_API_TOKEN=cfut_... && wrangler pages deploy . --project-name=provenlift --branch=main --commit-dirty=true

部署完打开 www.provenlift.com 验证线上效果和本地一致。

4

提交代码存档 MacBook

cd ~/Projects/gym-quote-page && git add . && git commit -m '修复排版问题' && git push
✓

完成！关掉 live-server 终端，合盖休息

客户网站在云端运行，不需要你的电脑开着。

终端什么时候能关？

原则：正在调试 → 保持终端开着。调试完并且代码已部署/同步 → 可以关闭。
live-server :8080 调客户页面时开着，部署到云端后关掉 部署后可关
Flask :5050（MacBook 本地） 调后台功能时开着，代码推到 Mac mini 后关掉 同步后可关
Flask（Mac mini 上） 系统自动运行，不需要你开终端，不需要你管 自动管理
cloudflared（Mac mini 上） 系统自动运行，让 crm.provenlift.com 能访问 自动管理
cloudflared（MacBook 上） 已停用。不要在 MacBook 上开隧道，会跟 Mac mini 冲突 不要开

常见问题

AI 找不到我的网页 / 打开的是乱码旧页面？

你的客户页面路径是：pkg/客户slug/latest.html

目前已有的客户方案：

· pkg/my-jason-400m/latest.html — 最新方案

· pkg/cpjason77098/latest.html

· pkg/plan001/latest.html

不要打开 pkg/400a.html，那是早期旧版，253KB 的大文件，内容已过时。

告诉 AI 这句话："最新客户页面在 pkg/my-jason-400m/latest.html，本地预览用 localhost:8080，线上是 www.provenlift.com。pkg/400a.html 是旧版不要打开。"

线上效果和本地预览不一致？

最常见原因：改了文件但忘了部署。跑一遍 wrangler deploy 命令就好。

第二常见：浏览器缓存。按 Ctrl+Shift+R（Mac 上是 Cmd+Shift+R）强制刷新。

极少见：本地改了 CSS/JS 但没保存。检查编辑器里文件是否已保存。

crm.provenlift.com 打不开？

用 ToDesk 连 Mac mini 检查服务是否在跑：

pgrep -fl cloudflared && lsof -i :5050

如果没有进程，重新加载服务：

launchctl load ~/Library/LaunchAgents/com.provenlift.flask.plist && launchctl load ~/Library/LaunchAgents/com.provenlift.tunnel.plist

Mac mini 必须已登录到桌面（不能停在登录界面），服务才会自动运行。

客户页面内容没更新？

① 如果改了模板/数据 → 先跑 python3 build/generator.py 重新生成

② 然后跑 wrangler deploy 部署到线上

③ 浏览器 Cmd+Shift+R 强制刷新确认
