两种本地预览，别搞混

调不同的东西，要开不同的终端

终端 A：live-server（端口 8080）MacBook 预览客户看到的报价页面
看的是 pkg/ 目录下的 HTML 文件
启动命令：live-server --port=8080
浏览器打开：localhost:8080/pkg/my-jason-400m/latest.html
什么时候开：改样式、改排版、改模板的时候
终端 B：Flask（端口 5050）MacBook 预览业务员后台 / 工作台
看的是你现在正在看的这个页面
启动命令：python3 build/admin_server.py
浏览器打开：localhost:5050
什么时候开：改后台功能、改工作台页面的时候
两个互不影响。你可以只开一个，也可以同时开两个，取决于你在调什么。

改完文件后的完整操作流程

两种情况，看你改的是什么文件

✅ 改样式 / 改交互（不用跑生成命令） MacBook
改了这些文件，保存后浏览器自动刷新：
· static/style.css（样式：颜色、字体、间距）
· static/*.js（交互：3D场景、详情弹窗、语言切换）
完整操作步骤：
第 1 步 开终端，启动本地预览
cd ~/Projects/gym-quote-page && live-server --port=8080 --host=0.0.0.0 第 2 步 浏览器打开预览页面
localhost:8080/pkg/my-jason-400m/latest.html 第 3 步 用编辑器改 CSS 或 JS，保存后浏览器自动刷新看效果，反复调到满意

第 4 步 满意后，部署到云端让客户看到
cd ~/Projects/gym-quote-page && export CLOUDFLARE_API_TOKEN=cfut_Je7kKN20exJ18UIWmpysZYjm016MSllBuR7qhTemc91a9b72 && wrangler pages deploy . --project-name=provenlift --branch=main --commit-dirty=true 第 5 步 代码存档到 GitHub
cd ~/Projects/gym-quote-page && git add . && git commit -m '样式调整' && git push 第 6 步 关掉 live-server 终端，完成。不需要碰 Mac mini。
⚠️ 改模板骨架 / 数据逻辑（需要重新生成） MacBook + Mac mini
改了这些文件，需要跑生成命令才能看到效果：
· build/templates/base.html（模板骨架）
· build/transform.py（数据处理逻辑）
· 飞书里的产品数据
完整操作步骤：
第 1 步 开终端，启动本地预览 MacBook
cd ~/Projects/gym-quote-page && live-server --port=8080 --host=0.0.0.0 第 2 步 用编辑器改模板或数据处理文件 MacBook

第 3 步 每次改完跑一次生成命令 MacBook
cd ~/Projects/gym-quote-page && python3 build/generator.py 浏览器刷新 localhost:8080/pkg/my-jason-400m/latest.html 看效果，反复调到满意

第 4 步 满意后推代码到 GitHub MacBook
cd ~/Projects/gym-quote-page && git add . && git commit -m '模板结构调整' && git push 第 5 步 ToDesk 连 Mac mini，拉最新代码 Mac mini
cd ~/Projects/gym-quote-page && git pull --rebase 第 6 步 重启 Mac mini 后台服务 Mac mini
launchctl unload ~/Library/LaunchAgents/com.provenlift.flask.plist && launchctl load ~/Library/LaunchAgents/com.provenlift.flask.plist 第 7 步 打开业务员后台，重新生成并部署 crm.provenlift.com
业务员后台 → 选中客户方案 → 点「生成预览」→ 确认效果 → 点「发给客户」部署上线
每个需要更新的客户方案都要重新生成一次

第 8 步 关掉 live-server，Mac mini 终端也可以关，完成。
💡 如果你改的是后台程序或后台页面：
即 build/admin_server.py 或 build/admin_templates/*
只需要做第 4、5、6 步（推代码 → Mac mini 拉代码 → 重启服务），不需要做第 7 步的重新生成。
因为后台页面不是生成出来的，重启 Flask 就直接生效了。
怎么判断走哪条路？看你改的是 static/ 目录下的文件还是 build/ 目录下的文件。改 static/ 走左边（简单），改 build/ 走右边（多几步）。如果两边都改了，先走右边把模板同步到 Mac mini，再走左边部署到云端。
